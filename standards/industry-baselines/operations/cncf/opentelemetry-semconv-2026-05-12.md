# OpenTelemetry semantic conventions 1.41.0

The Semantic Conventions define a common set of (semantic) attributes which provide meaning to data when collecting, producing and consuming it.
The Semantic Conventions specify among other things span names and kind, metric instruments and units as well as attribute names, types, meaning and valid values. For a detailed definition of the Semantic Conventions’ scope see [Semantic Conventions Stability](https://opentelemetry.io/docs/specs/otel/versioning-and-stability/#semantic-conventions-stability).
The benefit to using Semantic Conventions is in following a common naming scheme that can be standardized across a codebase, libraries, and platforms. This allows easier correlation and consumption of data.

Semantic Conventions are defined for the following areas:

- **[General](https://opentelemetry.io/docs/specs/semconv/general/): General Semantic Conventions**.
- [CICD](https://opentelemetry.io/docs/specs/semconv/cicd/): Semantic Conventions for CICD systems.
- [Cloud Providers](https://opentelemetry.io/docs/specs/semconv/cloud-providers/): Semantic Conventions for cloud providers libraries.
- [CloudEvents](https://opentelemetry.io/docs/specs/semconv/cloudevents/): Semantic Conventions for the CloudEvents specification.
- [Database](https://opentelemetry.io/docs/specs/semconv/db/): Semantic Conventions for database operations.
- [Exceptions](https://opentelemetry.io/docs/specs/semconv/exceptions/): Semantic Conventions for exceptions.
- [FaaS](https://opentelemetry.io/docs/specs/semconv/faas/): Semantic Conventions for Function as a Service (FaaS) operations.
- [Feature Flags](https://opentelemetry.io/docs/specs/semconv/feature-flags/): Semantic Conventions for feature flag evaluations.
- [Generative AI](https://opentelemetry.io/docs/specs/semconv/gen-ai/): Semantic Conventions for generative AI (LLM, etc.) operations.
- [GraphQL](https://opentelemetry.io/docs/specs/semconv/graphql/): Semantic Conventions for GraphQL implementations.
- [HTTP](https://opentelemetry.io/docs/specs/semconv/http/): Semantic Conventions for HTTP client and server operations.
- [Messaging](https://opentelemetry.io/docs/specs/semconv/messaging/): Semantic Conventions for messaging operations and systems.
- [Object Stores](https://opentelemetry.io/docs/specs/semconv/object-stores/): Semantic Conventions for object stores operations.
- [RPC](https://opentelemetry.io/docs/specs/semconv/rpc/): Semantic Conventions for RPC client and server operations.
- [System](https://opentelemetry.io/docs/specs/semconv/system/): System Semantic Conventions.

Semantic Conventions by signals:

- [Events](https://opentelemetry.io/docs/specs/semconv/general/events/): Semantic Conventions for event data.
- [Logs](https://opentelemetry.io/docs/specs/semconv/general/logs/): Semantic Conventions for logs data.
- [Metrics](https://opentelemetry.io/docs/specs/semconv/general/metrics/): Semantic Conventions for metrics.
- [Profiles](https://opentelemetry.io/docs/specs/semconv/general/profiles/): Semantic Conventions for profiles.
- [Resource](https://opentelemetry.io/docs/specs/semconv/resource/): Semantic Conventions for resources.
- [Trace](https://opentelemetry.io/docs/specs/semconv/general/trace/): Semantic Conventions for traces and spans.

Configuration:

- [Configuration](https://opentelemetry.io/docs/specs/semconv/configuration/): Semantic convention related configuration options.
