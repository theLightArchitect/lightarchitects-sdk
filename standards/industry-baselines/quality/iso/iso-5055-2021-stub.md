## Publicly-known scope

ISO/IEC 5055:2021 defines automated, source-code-level measures for four characteristics, drawn from CISQ's ASCQM (Automated Source Code Quality Measures):

| Characteristic | What's measured | Examples |
|----------------|------------------|----------|
| **Reliability** | code patterns associated with operational failure | unhandled exceptions, null dereferences, resource leaks |
| **Performance Efficiency** | code patterns associated with performance issues | inefficient algorithms, blocking I/O in hot paths |
| **Security** | code patterns associated with security vulnerabilities | injection vectors, weak crypto, hardcoded secrets |
| **Maintainability** | code patterns associated with maintenance difficulty | high complexity, code duplication, dead code |

The standard specifies **specific weakness types per characteristic** that automated tools must detect to claim ISO/IEC 5055 conformance. CISQ publishes complementary public materials at https://www.it-cisq.org/standards/ that describe the framework structure (without the verbatim normative text).
