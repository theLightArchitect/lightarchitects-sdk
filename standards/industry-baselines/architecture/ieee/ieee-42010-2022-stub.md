## Scope (from public abstract)

The standard establishes requirements for **architecture description (AD)** structure and expression across diverse entities including:
"software, systems, enterprises, systems of systems, families of systems, products (goods or services), product lines, service lines, technologies and business domains."

## Key concepts (from publicly-available summary; full normative text in paid standard)

The 42010 conceptual model defines:
- **Architecture description (AD)** — the artifact expressing an architecture
- **Architecture description framework (ADF)** — conventions for ADs in a domain
- **Architecture description language (ADL)** — language for expressing architectures
- **Stakeholder** — individual/team/organization with interests in the system
- **Concern** — interests held by stakeholders
- **Architecture viewpoint** — conventions for constructing one type of view
- **Architecture view** — expression of the architecture from one viewpoint
- **Architecture model** — fundamental component of a view
- **Model kind** — conventions for one type of model
- **Correspondence** — relations between AD elements
- **Correspondence rule** — rule asserting required correspondences

The 2022 revision distinguishes between **architectures** (the abstract structure) and **architecture descriptions** (the concrete artifact). It defines conformance requirements for ADs, ADFs, ADLs, viewpoints, and model kinds, but **does not prescribe** specific processes, methods, notations, techniques, tools, or recording formats.
