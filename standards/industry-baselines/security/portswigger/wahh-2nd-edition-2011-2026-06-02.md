## Notes on Age

Published 2011. Web stack has evolved (SPAs, GraphQL, JWT, OAuth flows). However:
- The **methodology** (Ch. 21) is timeless — still cited by current OWASP WSTG
- The **vulnerability classes** (Ch. 6-13) remain valid; modern frameworks change syntax but injection/XSS/CSRF semantics persist
- For **current/modern attacks** (GraphQL injection, OAuth2 misconfiguration, JWT confusion, SSRF in cloud meta, prototype pollution), supplement with OWASP WSTG v4.2 (`testing/owasp/`) and recent academic papers in `security/agentic-academic/`

WAHH remains canonical for **methodology + attack pattern catalogue**. Pair with current sources for modern stack specifics.
