```bash

make doctest        # doc-example tests (separate gate)

make push           # quality + git push

```

### Gateway Build Workaround

`lightarchitects-gateway` is excluded from the workspace (`Cargo.toml` `exclude` list) due to
worktree lockfile collisions. `make deploy` always fails with:
> "package ID specification `lightarchitects-gateway` did not match any packages"

**Fix**: temporarily add `"lightarchitects-gateway"` to `members` and remove from `exclude`,
run `cargo build --release -p lightarchitects-gateway`, copy binary, then **revert** `Cargo.toml`.
Do NOT commit the temporary workspace change.

## Coding Standards

Canonical: `~/lightarchitects/soul/helix/user/standards/builders-cookbook.md`

- NO `.unwrap()` / `.expect()` in production ‚Äî `clippy::unwrap_used` + `expect_used` (deny)
- NO `panic!()` ‚Äî `clippy::panic` (deny)
- `unsafe` requires `// SAFETY:` comment ‚Äî `unsafe_code` (deny)
- `clippy::pedantic` as errors
- All public items require doc comments (`missing_docs = "deny"`)
- Cyclomatic complexity ‚â§ 10, functions ‚â§ 60 lines
- Tests override via `#[allow(clippy::unwrap_used, clippy::expect_used)]`

## Key Design Decisions

| Decision | Choice |
|----------|--------|
| Span types | Owned by `lightarchitects::ayin::span` ‚Äî AYIN binary reads same JSON on disk |
| Client pattern | Two-level builder (octocrab-style): client construction vs per-call |
| Default transport | `HttpTransport` ‚Äî `XyzClient::builder().api_key("la_...").build()` hits `api.lightarchitects.ai` |
| Local/dev transport | `StdioTransport` ‚Äî `XyzClient::local_builder().binary_path(...).build().await` spawns binary |
| EVA transport | Dual-path: orchestrator adapter + 9 typed tool methods |
| SERAPH | Typed client only ‚Äî all offensive logic lives on the cloud gateway |
| Error hierarchy | `thiserror`, domain errors, no HTTP status codes |
| Retry | `TransportError` only ‚Äî tool errors are not transient |
| Auth | API key via `HttpTransport::builder().api_key()`; pre-spawn check for local path |

## Webshell UI Testing

```bash
# E2E suite ‚Äî always override to dev server (suite defaults to :8733 = production binary)
PLAYWRIGHT_BASE_URL=http://localhost:5173 pnpm exec playwright test e2e/webshell.spec.ts

# Regenerate visual baselines ‚Äî requires live dev server; run pnpm dev first
PLAYWRIGHT_BASE_URL=http://localhost:5173 pnpm exec playwright test e2e/webshell.spec.ts --update-snapshots -g "Visual regression"
```

**Gotchas:**
- Snapshot baselines regenerated without a running dev server are solid magenta (`#FF00FF`) ‚Äî always verify `pnpm dev` is up on :5173 before `--update-snapshots`.
- After any route rename: `grep -n 'location.hash' e2e/webshell.spec.ts` to find stale hash navigations in the E2E spec before running the suite.
